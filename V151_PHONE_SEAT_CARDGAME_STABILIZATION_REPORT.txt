Rummy 500 V151 — Phone Seat Cardgame Stabilization

Baseline
- Built from rummy500-v150-full-cardgame-alpha.zip.
- Preserved the existing saved-game library, shared-room URL behaviour, cloud sync, manual scoring, active player/presence display, and v149 persistence fixes.

What changed
1. Added per-device player claiming
- New "This phone" selector inside the playable cardgame panel.
- Each phone can be set to Table mode or claimed as a specific player.
- The selected seat is stored locally per game/room, so one phone can stay as You, another as GF, etc.

2. Safer multiplayer card handling
- If a phone is claimed as a player, draw/meld/lay-off/discard actions only work on that player's turn.
- When it is not your turn, the app shows that you are waiting for the current player.
- Table mode still works like the original v150 shared-screen alpha, where the current player's hand is shown and playable.

3. Hand display polish
- Added a small hand-owner line so it is always clear whose cards are being shown.
- Claimed phones show that player's hand; table mode shows the current turn player's hand.
- Card selection is cleared when changing the phone/player seat to avoid accidental actions.

4. State-safety fix
- Removed a direct mutation inside the stock-rebuild flow when the stock pile is empty and the discard pile is reshuffled.
- The card round now updates stock and discard piles immutably, which is safer for React/Supabase syncing.

Validation
- npm install completed in the sandbox, with an expected warning because the project requests Node 24.x while the sandbox uses Node 22.16.0.
- npx tsc --noEmit passed.
- next build compiled successfully, then hit the same sandbox Next.js worker EPIPE during the later lint/type-validation worker phase. This appears to be an environment/process issue after compilation rather than a source compile error.

Suggested next version
- V152 should improve discard-pile rules: let a player take a selected discard card plus everything above it, then require the picked card to be used in a meld/lay-off that turn. That would make the playable layer closer to real Rummy 500 rules.
